# C3 : Product

## 3.1 Development

A aplicação FestivAll foi desenvolvida como uma **Single Page Application (SPA)** em ReactJS, seguindo uma arquitetura de componentes funcional com gestão de estado baseada em React Hooks e Context API.

### Arquitetura geral

A aplicação está organizada em camadas independentes, cada uma com responsabilidades bem definidas:

- **Camada de apresentação** — componentes React (páginas e componentes reutilizáveis) que constroem a interface visual
- **Camada de estado** — Context API com dois providers (`AuthContext` e `FavoritesContext`) que gerem estado partilhado entre páginas, com persistência no `localStorage`
- **Camada de routing** — React Router v6 com rotas protegidas via componente `RequireAuth`
- **Camada de acesso a dados** — módulo `api/ticketmaster.js` que abstrai todas as chamadas HTTP à API Ticketmaster usando `fetch` nativo do browser
- **Camada de estilo** — Material UI com tema customizado (`theme.js`), CSS modular e biblioteca Framer Motion para animações

### Fluxo de dados

1. O utilizador insere a **Consumer Key** na página de Login
2. O `AuthContext` valida e guarda a chave no `localStorage`
3. O `RequireAuth` permite o acesso às rotas protegidas
4. Cada página chama funções do módulo `api/ticketmaster.js` via `useEffect`
5. Os dados são guardados em estado local com `useState` e renderizados
6. As ações do utilizador (favoritar, navegar) atualizam o estado e re-renderizam a UI

## 3.2 Instalation

### Pré-requisitos

- **Node.js 20+** (para desenvolvimento local)
- **Docker** e **Docker Compose** (para execução containerizada)
- Conta no portal de developers da Ticketmaster para obter Consumer Key gratuita

### Obtenção da Consumer Key

1. Aceder a [developer.ticketmaster.com](https://developer.ticketmaster.com)
2. Criar conta gratuita e fazer login
3. Copiar a **Consumer Key** disponível no dashboard

### Configuração do ambiente

Criar ficheiro `.env` na raiz do projeto a partir do `.env.example`:

```bash
VITE_TM_API_KEY=a_tua_consumer_key_aqui
```

### Execução com Docker (recomendado)

```bash
# Construir a imagem e arrancar o container
docker compose up --build

# Aceder à aplicação
# http://localhost:3000
```

Para parar:

```bash
docker compose down
```

### Execução local (sem Docker)

```bash
# Instalar dependências
npm install

# Arrancar o servidor de desenvolvimento
npm run dev

# Aceder à aplicação
# http://localhost:5173
```

Para construir para produção:

```bash
npm run build
npm run preview
```

### Configuração do Docker

O `Dockerfile` usa build multistage para otimizar o tamanho final da imagem:

```dockerfile
# Build stage — Node 20 compila o código com Vite
FROM node:20-alpine AS build
WORKDIR /app
ARG VITE_TM_API_KEY
ENV VITE_TM_API_KEY=$VITE_TM_API_KEY
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

# Production stage — Nginx serve os ficheiros estáticos
FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

A configuração do Nginx redireciona todas as rotas para o `index.html`, permitindo que o React Router gira a navegação client-side:

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

## 3.3 Usage

### Acesso inicial

Ao abrir a aplicação pela primeira vez, o utilizador é redirecionado automaticamente para a página de **Login**. É necessário fornecer uma Consumer Key válida da Ticketmaster para aceder ao restante conteúdo.

### Navegação

A barra de navegação superior (`NavBar`) está disponível em todas as páginas e dá acesso a:

- **Dashboard** — visão geral com eventos em destaque
- **Eventos** — pesquisa avançada com filtros
- **Artistas** — pesquisa de attractions
- **Favoritos** — eventos guardados pelo utilizador
- **Sair** — limpa a chave e devolve à página de login

### Limitações

- A aplicação depende da disponibilidade da API Ticketmaster — se a API estiver fora de serviço, não há dados
- A Consumer Key tem um limite de **5000 chamadas por dia** no plano gratuito
- Os favoritos são guardados localmente no browser (não há sincronização entre dispositivos)
- A pesquisa retorna no máximo 20 resultados por página

## 3.4 Implementation details

### Objetivos mínimos cumpridos

**Aplicação cliente em ReactJS**
A aplicação foi totalmente desenvolvida com React 18, utilizando exclusivamente componentes funcionais e React Hooks (useState, useEffect, useContext, useMemo). Foram criados 15 componentes próprios — 6 páginas, 4 componentes reutilizáveis, 2 context providers, mais os componentes `App`, `AppShell` e `RequireAuth`.

**Consumo de API REST com autenticação**
Todas as chamadas à API Ticketmaster são feitas via `fetch` no módulo `api/ticketmaster.js`, com a Consumer Key injetada automaticamente. A autenticação é validada antes de cada navegação através do componente `RequireAuth`.

**Containerização com Docker**
A aplicação é totalmente containerizada com build multistage (Node 20 Alpine para o build, Nginx Alpine para servir), reduzindo a imagem final para ~25 MB. O `docker-compose.yml` orquestra o serviço e injeta a Consumer Key como build argument do Vite.

**Submissão via GitHub e DockerHub**
Código-fonte publicado em organização do GitHub. Imagem Docker disponibilizada no DockerHub para execução simplificada.

### Elementos de valorização

**Material UI integrado em profundidade**
Para além do tema customizado, foram migrados componentes-chave para usar a biblioteca MUI:
- **Login** — `Paper`, `TextField`, `Button`, `Typography`, `Box`
- **NavBar** — `AppBar`, `Toolbar`, `Button`, `Box`
- **EventCard** — `Card`, `CardMedia`, `CardContent`, `Typography`, `Button`, `IconButton`, `Box`, `Chip`
- **SearchBar** — `TextField`, `Select`, `MenuItem`, `FormControl`, `InputLabel`, `Button`, `Box`

**Animações declarativas com Framer Motion**
Foram adicionadas três animações para melhorar a experiência de utilizador:
- Animação de **pop** no painel de login ao montar a página
- Animação de **scale com efeito spring** nos cards de eventos em hover
- Animação de **sublinhado expansivo** nos links da navbar em hover (CSS puro)

**Gestão de estado em três camadas**
- `useState` local em cada componente para estado efémero
- Context API com `AuthContext` e `FavoritesContext` para estado partilhado
- `localStorage` para persistência entre sessões

**Routing protegido**
Implementação de `RequireAuth` como Higher-Order Component que verifica o estado de autenticação e redireciona automaticamente para `/login` em caso de acesso não autorizado.

**Responsividade**
A interface adapta-se a três breakpoints principais (mobile, tablet, desktop) com CSS Grid e media queries.

**Configuração Nginx para SPA**
Configuração específica do Nginx para suportar client-side routing do React Router, com fallback para `index.html` em todas as rotas e cache otimizado para assets estáticos.
