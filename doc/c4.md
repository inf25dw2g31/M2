# C4 : Presentation

## Apresentação

A apresentação do projeto será realizada presencialmente com demonstração ao vivo da aplicação FestivAll completa em execução.

---

## Uso de Inteligência Artificial (transparência)

Este projeto foi construído com **auxílio significativo de Inteligência Artificial**, nomeadamente o assistente **Claude** (Anthropic). O grupo optou por usar a IA como ferramenta de apoio ao desenvolvimento, mantendo total responsabilidade pelas decisões, integração e validação do código final.

### Como a IA foi usada

**Apoio na compreensão de conceitos**
A IA foi consultada para esclarecer e explicar conceitos teóricos de React (componentes funcionais, hooks, Context API, ciclo de vida), Material UI (sistema de temas, prop `sx`, componentes Paper/Card/TextField), Framer Motion (animações declarativas, spring transitions), Docker (multistage builds, compose, volumes) e arquitetura web (three-tier, JWT, REST).

**Geração e refatoração de código**
A IA gerou trechos de código para os componentes em Material UI (Login, NavBar, EventCard, SearchBar), as animações com Framer Motion, os ficheiros Docker (Dockerfile, nginx.conf, docker-compose.yml) e os styles `sx` personalizados. Cada trecho foi revisto, adaptado ao projeto específico e integrado pelo grupo.

**Resolução de dúvidas e debugging**
Durante o desenvolvimento, a IA foi usada para identificar problemas (labels desalinhadas no MUI, cores herdadas do CSS global, configuração do Vite para variáveis de ambiente no Docker), sugerir soluções e explicar a causa raiz dos bugs.

**Documentação**
Os capítulos deste relatório (c1, c2, c3, c4), o README do repositório e a apresentação em PowerPoint foram preparados com apoio da IA, mantendo a revisão e validação humana de todo o conteúdo.

### Decisões mantidas pelo grupo

- Escolha da API externa (Ticketmaster Discovery API)
- Decisão de implementar backend próprio com Node.js + MySQL (em vez de manter apenas localStorage)
- Estrutura geral do projeto e organização de pastas
- Funcionalidades a incluir (favoritos, histórico, listas, avaliações)
- Design visual e paleta de cores
- Esquema da base de dados
- Estratégia de segurança (JWT + bcrypt + encriptação)
- Decisão de não usar a API do M1 (foco no React era prioritário)

### Postura do grupo perante o uso de IA

O grupo entende a IA como um colega de equipa virtual — útil para acelerar tarefas técnicas, esclarecer dúvidas e gerar ideias, mas que não substitui o trabalho de compreensão, integração, teste e responsabilidade do trabalho. Toda a aplicação foi testada manualmente, todas as decisões arquiteturais foram debatidas e todos os ficheiros foram revistos antes de submissão.

Esta abordagem alinha-se com as boas práticas atuais de uso ético de IA em contexto académico, onde a transparência sobre o auxílio recebido é mais importante do que escondê-lo.
