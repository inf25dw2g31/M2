# C4 : Presentation

## Apresentação

A apresentação do projeto será realizada presencialmente, com demonstração ao vivo da aplicação FestivAll em execução. A sessão cobrirá:

1. **Demonstração funcional** — execução da aplicação via Docker, login com Consumer Key, navegação pelas várias páginas (Dashboard, Eventos, Artistas, Favoritos) e demonstração das funcionalidades de pesquisa e favoritos
2. **Walkthrough técnico** — explicação da estrutura de pastas, componentes React, fluxo de dados entre Context API, hooks e camada de API
3. **Discussão da arquitetura** — escolhas tecnológicas (React + Vite + MUI + Framer Motion), padrões de design (componentes funcionais, HOC para rotas protegidas), containerização com Docker multistage
4. **Demonstração da camada de autenticação** — RequireAuth, AuthContext, validação da Consumer Key
5. **Perguntas e respostas**

## Uso de Inteligência Artificial

No desenvolvimento deste projeto foi utilizado o assistente de IA **Claude** (Anthropic) como ferramenta de apoio, de forma transparente. O uso da IA centrou-se nas seguintes tarefas:

- **Esclarecimento de conceitos teóricos** sobre React (componentes, hooks, Context API, ciclo de vida) com base nos slides da unidade curricular
- **Revisão e refatoração de código** dos componentes desenvolvidos, com foco na adoção de componentes Material UI e Framer Motion
- **Apoio na configuração do Docker** — criação do Dockerfile multistage, docker-compose.yml e configuração do Nginx para SPA routing
- **Resolução de dúvidas pontuais** sobre sintaxe, propriedades de componentes MUI e boas práticas

Toda a integração e validação do código foi feita pelo grupo. As decisões de arquitetura, escolha de funcionalidades, design da interface e estrutura do projeto foram decididas pelos membros do grupo. A IA atuou como assistente — equivalente a consultar documentação ou colegas — e não como substituto do trabalho realizado.

Esta transparência é alinhada com as boas práticas de uso ético de ferramentas de IA em contexto académico.
