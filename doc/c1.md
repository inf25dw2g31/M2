# C1 : FestivAll — Aplicação Web Cliente em ReactJS

## Description

O FestivAll é uma Aplicação Web fullstack desenvolvida no âmbito do segundo momento (M2) de avaliação contínua da unidade curricular de Web Development 2. A aplicação foi pensada como uma plataforma de descoberta cultural e desportiva, permitindo ao utilizador pesquisar eventos em tempo real através da **Ticketmaster Discovery API**, guardar favoritos, manter histórico, criar listas personalizadas, atribuir avaliações e gerir o seu perfil.

A arquitetura é composta por três camadas independentes:

**Frontend (Cliente React)**
Construído com **React 18** e **Vite**, com interface baseada em **Material UI** e animações com **Framer Motion**. A navegação SPA é gerida pelo **React Router v6**, com proteção de rotas através do componente `RequireAuth`. O estado partilhado é mantido pelo Context API (`AuthContext` e `FavoritesContext`).

**Backend (API REST própria)**
Construído com **Node.js + Express**, expõe endpoints autenticados por **JWT** com tokens emitidos a cada login. As passwords são guardadas com hash **bcrypt** e a Consumer Key da Ticketmaster é **encriptada** antes de ser persistida na base de dados. Inclui **rate limiting** para prevenir abuso e middleware de autorização que valida o token em cada pedido protegido.

**Base de dados (MySQL)**
Esquema relacional com **7 tabelas** (users, favorites, history, saved_searches, ratings, lists, list_events) com chaves estrangeiras, restrições de unicidade e cascade delete. Dados persistentes através de volume Docker.

Toda a aplicação é orquestrada com **Docker Compose**, lançando os três serviços (frontend, API, base de dados) com um único comando.

**Funcionalidades principais:**
- Registo de utilizador com associação da Consumer Key da Ticketmaster
- Login com JWT e gestão de sessão
- Dashboard com eventos em destaque
- Pesquisa de eventos com filtros (palavra-chave, cidade, data, categoria)
- Detalhe de cada evento (artistas, local, datas, preços, link para bilheteira)
- Sistema de favoritos persistente por utilizador
- Histórico de eventos consultados
- Avaliações com estrelas e notas
- Listas personalizadas (estilo playlists)
- Perfil de utilizador com atualização da Consumer Key
