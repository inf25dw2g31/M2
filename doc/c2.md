# C2 : Resources

A aplicação consome a **Ticketmaster Discovery API v2**, uma API REST pública mantida pela Ticketmaster que disponibiliza informação sobre eventos culturais e desportivos em todo o mundo.

## Endpoint base

```
https://app.ticketmaster.com/discovery/v2
```

## Autenticação

Todos os pedidos exigem uma **Consumer Key** (API Key) registada no portal de developers da Ticketmaster ([developer.ticketmaster.com](https://developer.ticketmaster.com)). A chave é enviada como query parameter em cada pedido:

```
?apikey=CONSUMER_KEY
```

Sem chave válida, a API responde com `401 Unauthorized`.

## Recursos utilizados

### Events

Pesquisa e listagem de eventos.

| Método | Endpoint | Descrição |
|--------|----------|-----------|
| GET | `/events.json` | Lista eventos com filtros opcionais (keyword, city, date, classificationName, size) |
| GET | `/events/{id}.json` | Devolve o detalhe completo de um evento específico |

**Parâmetros usados:**
- `keyword` — pesquisa por nome de artista ou evento
- `city` — filtra por cidade
- `startDateTime` / `endDateTime` — intervalo de datas
- `classificationName` — categoria (Music, Theatre, Sports, etc.)
- `size` — número de resultados por página

### Attractions (Artistas)

Pesquisa de artistas, bandas e atrações.

| Método | Endpoint | Descrição |
|--------|----------|-----------|
| GET | `/attractions.json` | Lista artistas com filtros opcionais (keyword, size) |

## Estrutura dos dados

A resposta da API é em JSON e segue um modelo hierárquico:

```json
{
  "_embedded": {
    "events": [
      {
        "id": "G5vYZ9...",
        "name": "Coldplay — Music of the Spheres",
        "dates": { "start": { "localDate": "2026-07-12" } },
        "classifications": [{ "segment": { "name": "Music" } }],
        "_embedded": {
          "venues": [{ "name": "Estádio da Luz", "city": { "name": "Lisboa" } }],
          "attractions": [{ "name": "Coldplay" }]
        },
        "priceRanges": [{ "min": 45, "max": 180, "currency": "EUR" }],
        "images": [{ "url": "https://..." }]
      }
    ]
  }
}
```

## Limitações

- **Rate limit:** 5000 chamadas por dia por Consumer Key (plano gratuito)
- **Sem escrita:** API exclusivamente de leitura (GET) — não permite criar/editar eventos
- **Sem OAuth2:** a autenticação é apenas por API Key, sem identidade de utilizador final

## Camada de proteção na aplicação cliente

Embora a API utilize Consumer Key em vez de OAuth2/JWT, a aplicação implementa proteção de rotas no frontend:

- **`AuthContext`** — guarda a chave no `localStorage` e expõe `isAuthenticated`
- **`RequireAuth`** — Higher-Order Component que protege as rotas privadas e redireciona para `/login` quando não há chave
- **Camada API** — todas as funções em `src/api/ticketmaster.js` recebem a chave do contexto e enviam-na em cada pedido
